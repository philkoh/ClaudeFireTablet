User's project at `/home/phil/ClaudeFireTablet` does interactive Android development on a Fire HD 10 Plus 11th gen connected via USB. Package: `com.example.firehello`. **Current build v0.2 is confirmed working end-to-end** — user said "looks great" after seeing live video with 5-second delay on PC.

## What it does
Front camera → Camera2 capture → MediaCodec H.264 encoder (640×480, 30fps, 2 Mbps, I-frame every 1s) → `ServerSocket` on port 7777. PC connects via `adb forward`, buffers 5s of bytes in `stream.py`, pipes into `ffplay -f h264` for display.

## Run loop (each iteration)
```bash
cd /home/phil/ClaudeFireTablet
./build.sh                                                # ~3s
adb install -r build/app-debug.apk                        # if signing keeps stable
adb shell am start -n com.example.firehello/.MainActivity
# tablet shows status text; keep it foregrounded (camera releases on background)
adb forward tcp:7777 tcp:7777                             # once per adb daemon
python3 stream.py                                         # opens ffplay window
```

## Known gotchas
- **Keystore regeneration**: `build.sh` wipes `build/` on each run, so the debug keystore is regenerated. The first `adb install -r` after a build fails with sig mismatch — do `adb uninstall com.example.firehello` then `adb install` (no `-r`). Or: pull the keystore out of `build/` and keep it elsewhere.
- **Camera releases on background**: when the activity loses focus (home button, screen-off), `Camera2ClientBase: Closed Camera` fires and the encoder stops producing frames. Stream goes silent. Bring the app back to foreground to resume. `MainActivity` uses `FLAG_KEEP_SCREEN_ON` to mitigate auto-lock.
- **stream.py exits on player window close**: handled with `BrokenPipeError` catch (added after v0.2 confirmed). Closing the ffplay window now prints "player window closed" and exits cleanly.
