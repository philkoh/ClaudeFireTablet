When the user asks how to do something with their hardware/tools, give one definitive answer rather than presenting several possibilities. If uncertain, use web search to nail it down before answering.

**Why:** User explicitly pushed back twice during the ADB-enable walkthrough: "be specific rather than 'it might be this or it might be that'" and later "I don't see 'USB computer connection'… You're confusing me with 'maybe it's this, maybe it's that'." Hedging slows them down and forces them to do the research I should be doing.

**How to apply:** Before answering procedural questions about a specific device/version/tool, run web searches to find the authoritative answer. Then write a single concrete path. Save fallback troubleshooting for *after* the primary path, under a clearly-labeled "if that fails" section — not interleaved with the main steps.
